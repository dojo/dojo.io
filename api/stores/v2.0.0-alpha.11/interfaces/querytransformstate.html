fetchAroundUpdates

fetchAroundUpdates: boolean
  • Defined in src/store/mixins/createQueryTransformMixin.ts:66
  • Defined in src/store/createQueryTransformResult.ts:127

We have to track whether we're fetching around updates because the query transform results need to know this in order to determine whether they can fully trust the data provided to them from the source store's observabel Is the parent store fetching around updates If the parent store is fetching around updates, we will always have the latest superset of this view's data in the updates it receives locally. In that case, even if actively tracking, no additional fetches need to be performed, the local queries and transformations can just be applied to the new data directly.
